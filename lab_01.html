Preguntas a Resolver | HTML

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es la red global de computadoras conectadas, mientras que la World Wide Web es un sistema de información accesible a través de Internet.

2. ¿Cuáles son las partes de un URL?

Las partes de un URL incluyen el esquema (protocolo), dominio, puerto (opcional), ruta, consulta y fragmento.

3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET obtiene recursos, HEAD obtiene encabezados, POST envía datos, PUT reemplaza, PATCH modifica y DELETE borra.

4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se usa POST porque envía datos de manera segura sin mostrarlos en la URL.

5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se usa el método GET para solicitar y recibir el contenido de la página.

6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

El código 200 indica que la solicitud fue exitosa y no hubo error.

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Depende. Si la URL es incorrecta, no es culpa del desarrollador. Si la página fue eliminada sin redirección, sí.

8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, un error 500 indica un problema en el servidor, lo que generalmente es responsabilidad del desarrollador.

9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Significa que su uso ya no se recomienda. Ejemplos: <font>, <center>, <acronym>.

10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 introduce nuevas etiquetas semánticas, APIs nativas y elimina atributos de presentación.

11. ¿Qué componentes de estructura y estilo tiene una tabla?

Componentes: <table>, <thead>, <tbody>, <tfoot>, <tr>, <th>, <td>. Se estiliza con CSS.

12. ¿Cuáles son los principales controles de una forma HTML5?

Input types avanzados (email, date, color), etiquetas <datalist>, <output>, <progress>.

13. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas?

Tengo chrome y cuenta con soporte en su totalidad para HTML5 .

Preguntas a Responder | CSS

1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

Solo utilizarlos cuando se necesite forzar un estilo en casos específicos, ya que evita la sobrescritura de estilos

2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Porque afecta el rendimientoy la carga de la página y fondos con poco contraste pueden reducir la legibilidad del contenido.

3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

  • %: Útil para diseños responsivos, ajustándose al contenedor padre.
  • px: Bueno para precisión en elementos fijos, pero poco flexible.
  • pt: No recomendado en web, más usado en impresión.

    • 4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    Reduce el tamaño del archivo eliminando espacios y comentarios, acelerando la carga del sitio y mejorando el rendimiento.
